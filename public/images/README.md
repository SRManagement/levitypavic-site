# Adding your photos

Drop image files into this folder using these exact names. The site will
automatically pick them up — no code changes needed.

| Filename           | Used for                              | Suggested size       |
|--------------------|----------------------------------------|-----------------------|
| cover.jpg          | Large "unlock" card at the top         | 800×900 (portrait)    |
| portrait.jpg       | Polaroid profile card                  | 800×900 (portrait)    |
| link-instagram.jpg | Instagram row thumbnail                | 400×400 (square)      |
| link-tiktok.jpg    | TikTok row thumbnail                   | 400×400 (square)      |
| link-snapchat.jpg  | Snapchat row thumbnail                 | 400×400 (square)      |
| feed-1.jpg         | Feed grid, tile 1                      | 600×600 (square)      |
| feed-2.jpg         | Feed grid, tile 2                      | 600×600 (square)      |
| feed-3.jpg         | Feed grid, tile 3                      | 600×600 (square)      |
| feed-4.jpg         | Feed grid, tile 4                      | 600×600 (square)      |
| feed-5.jpg         | Feed grid, tile 5                      | 600×600 (square)      |
| feed-6.jpg         | Feed grid, tile 6                      | 600×600 (square)      |

Notes:
- .jpg, .jpeg, .png, or .webp all work — just keep the filename before the dot exactly as above (if you use a different extension, rename the file to end in `.jpg`, or ask to have the code path updated).
- If a file is missing, that slot shows a small placeholder instead of breaking the page — so you can add photos gradually.
- To update a photo later: just replace the file with the same name and re-upload it to your GitHub repo (or drag-and-drop the new version into GitHub's web UI). Vercel redeploys automatically within a minute or two.
